A-t-on le droit de tricher ?



Peut-on ...

modifier les jeux ?

Oui, c'est le but ! Les énigmes vous invitent à rentrer dans le fonctionnement du jeu et à le décortiquer, afin de trouver ce que vous devez faire pour obtenir le mot de passe secret. Un mur vous bloque ? Utilisez un éditeur pour l'enlever. Une fonction de vérification vous embête ? Modifier le code source et remplacez-la par "return true".

Attention, si vous modifiez un jeu, vous n'avez plus la garantie qu'il vous donnera son mot de passe. Heureusement, vous avez toujours la possibilité de revenir à la version originale du jeu, après l'avoir mieux compris, et d'y effectuer les actions adéquates.

réfléchir à plusieurs / demander de l'aide ?

Oui, mais les réponses et les indices doivent être transmis en privé. Vous pouvez par exemple demander de l'aide sur un forum ou un serveur Discord auquel tout le monde a accès, puis avoir une personne qui vous donne des informations par message privé.

À vous de déterminer ce qui est "publique" et "privé" sur Internet. (Ce n'est pas forcément simple).

brute-forcer ? (C'est à dire : essayer toutes les combinaisons possibles d'un mot de passe).

Oui, mais ce n'est pas la meilleure méthode. Cependant, vous pouvez mélanger plusieurs approches.

Par exemple, supposons un jeu qui semble avoir 10000 mots de passe possibles. Vous l'explorez un peu et vous arrivez à réduire ce nombre à 10. En explorant encore plus, vous pourriez trouver le bon, mais ça vous semble trop compliqué, alors vous essayez les 10, un par un.

Pour vous donner un ordre d'idée :

- Si un jeu semble avoir moins de 20 solutions possibles, vous pouvez les tester manuellement.

- Si un jeu semble avoir moins de 500 solutions, facilement vérifiables, vous pouvez écrire un petit algorithme dans le language que vous souhaitez pour les tester automatiquement.

- Si un jeu semble avoir plus de 500 solutions, vous devriez peut-être continuer de l'explorer pour mieux le comprendre.

Utiliser des IA ?

Oui, mais je trouverai ça dommage pour vous. Vous tireriez plus de satisfaction à résoudre une énigme sans l'aide d'un système qui vous mâche le travail à outrance.

Essayez d'utiliser l'IA de manière raisonnable, pour vous aider uniquement sur une partie précise et délimitée d'une énigme. Par exemple, pour comprendre un morceau de code spécifique.

Si je peux permettre une leçon de morale : utilisez raisonnablement l'IA quelles que soient les tâches que vous avez à faire. Essayez de ne pas trop dépendre d'entités aussi puissantes, imprévisibles et fermées.

Publier des solutions ou des indices ?

Non. Même en les protégeant avec une "spoiler alert".

Si je trouve des parties de solutions ou des mots de passe d'énigmes, je ferais mon possible (même si c'est peu) pour les faire supprimer.

Réutiliser les énigmes pour en fabriquer d'autres ?

Oui, mais vous devez protéger votre solution avec un procédé similaire à celui que j'utilise (un fichier chiffré avec le mot de passe secret de l'énigme). Le but est d'éviter que vos énigmes donnent des indices trop flagrants sur mes énigmes.

Explorer le contenu du repository pour chercher des solutions ?

Oui. Il est toujours possible de résoudre une énigme sans consulter le repository, mais vous pouvez quand même le faire.

Le repository est un historique contenant des points de sauvegarde successifs (des "commits"). Vous pourriez y trouver des parties d'énigmes non terminée, ce qui peut vous aider à comprendre son fonctionnement interne.
