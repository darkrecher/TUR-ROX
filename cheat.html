A-t-on le droit de tricher ?



Peut-on ...

modifier les jeux ?

Oui, c'est le but ! Les énigmes vous invitent à rentrer dans le jeu et à le décortiquer, afin de trouver ce que vous devez faire pour obtenir le mot de passe secret. Un mur vous bloque ? Utilisez un éditeur pour l'enlever. Une fonction de vérification vous embête ? Modifier le code source pour qu'elle ne vérifie plus rien.

Attention, si vous modifiez un jeu, vous n'avez plus la garantie qu'il vous donnera son mot de passe. Heureusement, vous avez toujours la possibilité de revenir à sa version originale, pour y effectuer les actions adéquates une fois que vous avez compris son fonctionnement.

réfléchir à plusieurs / demander de l'aide ?

Oui, mais les réponses et les indices doivent rester privés. Vous pouvez demander de l'aide sur un forum public ou un serveur Discord, mais les personnes qui vous répondront devront le faire par message privé.

À vous de déterminer ce qui est "publique" et "privé" sur Internet. (Ce n'est pas forcément simple).

brute-forcer ? (C'est à dire : essayer toutes les combinaisons possibles d'un mot de passe).

Oui, mais ce n'est pas la meilleure méthode. Cependant, vous pouvez mélanger plusieurs approches.

Par exemple, supposons un jeu qui semble avoir 10000 mots de passe possibles. Vous l'explorez un peu et vous arrivez à réduire ce nombre à 10. En explorant encore plus, vous pourriez trouver le bon mot de passe, mais ça vous semble trop compliqué, alors vous essayez les 10 un par un.

Pour vous donner un ordre d'idée :

- Si un jeu semble avoir moins de 20 solutions possibles, vous pouvez les tester manuellement.

- Si un jeu semble avoir moins de 500 solutions, facilement vérifiables, vous pouvez écrire un petit algorithme dans le language que vous souhaitez pour les tester automatiquement.

- Si un jeu semble avoir plus de 500 solutions, vous devriez peut-être continuer de l'explorer pour mieux le comprendre.

utiliser des IA ?

Oui, mais je trouverai ça dommage pour vous. Vous tireriez plus de satisfaction à résoudre une énigme sans l'aide d'un système qui vous mâche le travail à outrance.

Essayez d'utiliser l'IA de manière raisonnable, pour vous aider uniquement sur une partie précise et délimitée d'une énigme. Par exemple, pour comprendre un morceau de code spécifique.

Si je peux me permettre une leçon de morale : utilisez raisonnablement l'IA quelles que soient les tâches que vous avez à faire. Essayez de ne pas trop dépendre d'entités aussi puissantes, imprévisibles et fermées.

publier des solutions ou des indices ?

Non. Même en les protégeant avec une "spoiler alert".

Si je trouve des solutions ou des mots de passe disponibles publiquement, je ferais mon possible (même si c'est peu) pour les faire supprimer.

réutiliser les énigmes pour en fabriquer d'autres ?

Oui, mais vous devez protéger votre solution avec un procédé similaire à celui que j'utilise (un fichier chiffré avec le mot de passe de l'énigme). Le but est d'éviter que vos énigmes donnent des indices trop flagrants sur mes énigmes.

explorer le contenu des repositories pour chercher les solutions ?

Oui, même s'il est toujours possible de résoudre une énigme sans faire ça.

Le repository est un historique contenant des points de sauvegarde successifs (des "commits"). Vous y trouverez les énigmes à différentes étapes de leur création, ce qui pourrait vous aider à comprendre leur fonctionnement interne. C'est aussi une occasion de s'initier à l'outil git.
